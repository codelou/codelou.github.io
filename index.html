hexo+github博客搭建

前言

刚刚搭建完这个博客后我就迫不及待地来一展身手，想了那么多，还是决定分享一下如何来搭建这个博客网站，好了，废话不多说，上步骤。

环境准备

安装git

window的童鞋安装git Bash

mac的童鞋直接使用mac自带的git命令行就行了

安装node

说真的现在写项目(特别是前端)没有node根本就没法混，可以直接到官网下载或者输入命令行，详细的到官网看。node安装好了，再安装npm，它是一个node的包管理工具，非常强大，到后来我们也会用到的。

安装hexo

$ sudo npm install hexo-cli -g

使用hexo

#创建hexo文件夹

$ mkdir hexo

#进入hexo的文件夹里

$ cd hexo

#初始化hexo

$ hexo init

#安装依赖包

$ npm install

#本地查看，运行下面命令，然后再浏览器里输入localhost:4000即可

$ hexo server

#问题：执行hexo server 提示找不到命令

#解决方法：在hexo 3.0 后server被当初出来了，需要安装server，安装命令：

$ npm install hexo -server --save

生成静态网站

$ hexo generate

目录结构

├── .deploy #需要部署的文件

├── node_modules #Hexo插件

├── public #生成的静态网页文件

├── scaffolds #模板

├── source #博客正文和其他源文件，404、favicon、CNAME 都应该放在这里

| ├── _drafts #草稿

| └── _posts #文章

├── themes #主题

├── _config.yml #全局配置文件
└── package.json

全局配置_config.yml

进入_config.yml后，我们重要需要改变的地方是deploy，早到deploy后我们改为

#注意冒号后面要留一个空格

type: git

#地址就如'https://github.com/codelou/codelou.github.io.git'

repo: github创建地址.git

branch: master

使用GitHub

注册帐号

创建代码仓库

这个仓库的名字要和你的帐号对应。格式:yourname.github.io

编辑文章

#新建文章

$ hexo new "第一篇博客"

在_posts目录下就会生成文件第一篇博客.md

在里面编辑完后保存，预览

$ hexo server

hexo部署

执行下列指令完成部署

$ hexo generate

$ hexo deploy

以下提示说明部署成功

[info] Deploy done: git

注意事项：如果执行输出

ERROR Deployer not found: git

有这几种可能：

  1. 配置文件里冒号后面没有留空格
  2. 版本问题，有人可能用2.0或3.0

解决方案：

1.修改配置文件，保证冒号后面留空格
2.修改配置文件deploy里的type为git(原来就是git的略过)，修改完之后输入命令行
npm install hexo-deployer-git --save安装依赖
3.如果上面两个方法还不行，用sudo权限进行操作

终于搭建完第一个属于自己的博客了，也希望对大家有帮助。
